Lección 17 - Web Workers

Definción: El evento loop es como el sistema encargado de ejecutar diversos eventos realizados por el usuario, clicks en botones, selecionar texto, cargar un archivo, etc. El problema con esto es que el event loop solo puede encargase de un unico evento a la vez, es decir, no puede cargar un archivo y mientras sucede esto, ir rellenando in input. Para esto llegaron los web workers, quienes permiten ejecutar scripts en hilos de segundo pano.
Cabe destacar que el web worker tiene sus limitaciones no puede ejecutar tanto codigo como el script principal, tampoco puede colocar eventos, si no solo se encarga de funciones especificas y sencillas.

Creación Worker


const worker = new Worker('nombre del script.js'), Recibe como parametro el nombre del script que se ejecutara en segundo plano
Tipos de Webs Workers
  • Dedicated Worker
  • Shared Worker
  • Service Worker
  • Abstract Worker
Metodos:

  • postMessage():

    Envia mensajes entre el script principal y el Web Worker

  • terminate():

    Termina el worker
Dedicated Worker y Shared Worker van de la mano al igual que Service Worker y Abstract Worker.

Lección 2 - Modo Estricto

Definición: Es un modo que puede ser activado por el desarrollador el cual permite que el codigo este mejor organizado Caracteristicas:

  • Convierte errores de JavaScript en excepciones
  • Mejora la optimización de los errores y consigue mejores tiempos de ejecución
  • Evita sintaxis usadas anteriores a ES6
  • No permite que el desarrollador realice una "mala sintaxis"

Errores que no admite:

  • Variables no declaradas
  • defineProperty, es un metodo que permite agregar a un objeto una nueva propiedad, este metodo tiene un valor que le podemos asignar que es, writeable: true, el cual hace que la propiedad creada no pueda ser modificada.
  • preventExtensiones, es un metodo que no deja agregar mas propiedades a objeto, sin el "use strict", no dejar agregar mas propiedades pero no arroja ningun error, con el "use strict" activado nos arroja error.
  • Propieddes en String, sin "use strict", no nos salta ningun error aunque no sea permittido, con "use strict" si.
  • No permite que hayan dos parametros con el mismo nombre en una función, ejemplo "function prueba(texto, texto){}"
  • No permite nombrar una variable con palabras reservadas.
  • This ya no se permite en funciones.
  • arguments y eval no pueden ser nombres de variables.

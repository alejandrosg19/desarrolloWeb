Lección 11 - Posicion

Con esta propiedad, podemos cambiar el flujo de como HTML esta leyendo los elementos, un elemento esta posicionado si afecta el flujo HTML

Tipos de Posicion

Static: Es la propiedad por defecto de los navegadores, esto no posiciona a los elementos

Relative: Es una propiedad que permite mover el elemento a otras partes del DOM desde una posición fija, esta posición fija corresponde a la posición que se le dio en un principio al elemento en HTML, por ejemplo a continuación veremos como la posición de la Caja 2 se sigue guardando en el DOM, en ningún momento la Caja 3 sube y toma la posición de la Caja 2, simplemente la Caja2 se esta moviendo a partir de dicha posición conservada o guardada en el DOM. Ademas esta propiedad contiene 4 nuevas propiedades que pueden ser aplicadas

  • top: Mueve el elemento hacia abajo, es como si se le diera la cantidad px especificada de espacio en la parte superior del espacio guardado por el DOM, ejemplo top: 50px
  • righ: Mueve el elemento a la derecha, da los espacios en la derecha de la caja ejemplo righ: 50px
  • left: Mueve el elemento a la izquierda, da los espacios en la izquierda de la caja ejemplo left: 50px
  • bottom: Mueve el elemento hacia arriba, es como si se le diera la cantidad px especificada de espacio en la parte inferior del espacio guardado por el DOM, ejemplo bottom: 50px
  • z-index: Cunado existen dos elementos con positio: relative, una elemento se vaa posicionar por enciama del otro pero si queremos que el elemento que esta debajo quede arriba, simplemente le damos la propiedad z-index: 1; o cualquier valor esto posicionara al elemento por encima del otro.

Absolute: Funciona igual que el Relative, solo que posee tres diferencias

  • Diferencia 1: En el relative se observaba como al aplicar la propiedad en un elemento, los demas respetaban la posicion inicial de dicho elemento, es decir, en el DOM seguia todo intacto solo se aplicaba los movimientos que se le daban al elemento, en absolute no sucede asi, si no que ahora esa aposicion ya no esta mas reservada y los otros elementos pueden tomar dicha posición
  • Diferencia 2: La posición de referencia que tenia el elemento para moverse hacia los diferentes lados, ya no va a hacer el espacio reservado que tenia si no que ahora va a hacer el viewport, que corresponde a toda la ventana del navegador, sin importar que el elemento este dentro otro es decir, un div hijo que esta dentro de un div padre. En este ultimo caso, el elmento hijo puede tomar de referencia al elemento padre, solo si este tiene un posicion: relative; olvidandose del viewport
  • Diferencia 3: Si elemento no posee ningun ancho ni alto, los los elementos que eran en bloque y conservaban todo el ancho de la ventana, ahora este ancho quedara igual al tamaño del elemento, es decir se convierte a un inline-block

Fixed: Funciona igual que el Absolute, solo que posee una una diferencia de que queda fijado en la ventana en el viewport es decir, que aunque demos scroll el elemento se mantiene

Ejemplo posicion: fixed; -> Explicación mas abajo

Sticky: Este tipo de posicion es una combinación del relative y el fixed, es decir tabien se queda fijado pero a diferencia del fixed este si guarda su posición y no es necesario realizar trucos dandole padding al body para que no se tapen las letras si no que este ya respeta y guarda la posición. Hay que tener muy en cuenta si el div o el elemento que contiene la propiedad de sticky se en cuentra dentro de otro div, porque entonces el div hijo que es el que contiene la propiedad solo se quedara fijado dentro del rango del div o caja padre.

Ejemplo posicion: Sticky;
